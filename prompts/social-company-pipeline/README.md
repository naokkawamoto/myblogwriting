# 会社コンテンツ起点 → X / LinkedIn 下書きパイプライン

ブログ・NEWS・記事（HTML またはテキスト）と、YouTube（難しければディスクリプションのテキスト）を渡し、**テーマに沿った下書き**を作る。  
その後 **外部の別LLM** で評価・フィードバックを受け、それを渡して **ブラッシュアップ**し、最後に **人間がレビュー** する想定のプロンプト群です。

**投稿（X / LinkedIn への公開）は自動化しません。** 承認後に人間がコピー投稿する、または社内で決めたスケジューラに手で入れる運用を想定しています。API連携や予約投稿の仕組みは本フォルダの範囲外です。

## 使い方（4ステップ）

1. **`01_writer_draft.md`** をコピーし、プレースホルダを埋めてから執筆用LLMに貼る → **下書き**を得る。
2. 下書きと同じ入力の要約（テーマ・ソースの抜粋で可）を添え、**`02_external_evaluator.md`** を評価専用の **別LLM・別セッション** に貼る → **フィードバック**のみを得る（執筆LLMと同一だと甘くなりがちなので分離推奨）。
3. **`03_writer_refine.md`** に「初稿＋フィードバック＋必要ならソース再掲」を埋めて執筆用LLMに貼る → **改稿**を得る。
4. **`04_human_review_checklist.md`** で人間が最終確認してから投稿する。

## ファイル一覧

| ファイル | 役割 |
|----------|------|
| `01_writer_draft.md` | ソース＋テーマから X / LinkedIn 下書きを書かせる |
| `02_external_evaluator.md` | 外部LLM用・評価と改善指示のみ（本文の全面書き換えは禁止寄り） |
| `03_writer_refine.md` | フィードバックを反映した改稿 |
| `04_human_review_checklist.md` | 人間レビュー用チェックリスト |

## メンテ

- 文字数上限やトーンは `01` / `03` の指示を会社方針に合わせて調整する。
- note 記事の作成は従来どおり `prompts/note/` を使う（本パイプラインは「会社記事・YouTube起点」のSNS下書き向け）。
